Architecture Orientée Services

Service
  • C'est un composant logiciel (comme les objets et les agents).
  • Il peut effectuer un ensemble d'opérations bien définies.
  • Il offre ses services (fournisseur) et profite d'autres services (client).
Service
  • Il est unique (à l'inverse de l'objet).
  • Il est représenté par une interface publique (ensemble d'opérations).
  • Son implémentation est cachée aux clients (boite noire).
Architecture Orientée Services
(SOA, Service Oriented Architecture)
  • C'est un ensemble de services qui communiquent entre eux.
  • Ces services peuvent être localisés sur la même machine ou réparties sur un réseau de machines.
Exemple de l'architecture
Caractèristiques

Pour être effective une architecture SOA a besoin de :

  • Un annuaire pour publier, rechercher et localiser les services disponibles.
  • Un moyen pour decrire les interfaces.
  • Un moyen pour l'invocation d'un service.
  • Un format pour l'échange de données entre services.
  • Un moyen pour transporter les données.
Avantages
  • Interopérabilité.
  • Réutilisabilité.
  • Abstraction.
  • Découplage.
  • Plus les avantages des systèmes distribués.

Les Services Web

Définition

Ils implémentent une architecture SOA basée sur les langages et les protocoles ainsi que les infrastructures du web.

Technologies

Il existe deux types de services web :

  • Les services web de type REST.
  • Les services web WS-*.
Services Web RESTful
  • Basés principalement sur HTTP, URI, HTML, XML et JSON.
  • Respectent les contraintes imposées par REST.
  • Ne permettent que certains types d'opérations (CRUD).
  • La manipulation de ressources et leurs unique objectif.
Services Web RESTful

Les ressources sont :

  • Localisées par une URI.
  • Représentées en HTML, XML, ou JSON, ...
  • Manipulées à l'aide de requêtes HTTP (POST, GET, PUT, DELETE).
Services Web WS-*
  • Permettent tout type d'opérations.
  • Reposent sur les standards SOAP, WSDL, UDDI, HTTP, XML, JSON, ...
SOAP
(Simple Object Access Protocol)
  • Protocole qui permet la transmission de messages entre objets.
  • Bâti sur XML.
  • Son rôle est l'invocation des services.
  • Peut être utilisé avec n'importe quel protocole de transmission (HTTP, SMTP, ...).
WSDL
(Web Services Description Language)
  • Langage basé sur XML.
  • Utilisé pour décrire l'interface d'un service.
  • Une description spécifie les opérations possibles, le format de données à utiliser pour communiquer, ce que renvoi les opérations, ...etc
UDDI
(Universal Description Discovery and Integration)
  • Annuaire de services fondé sur XML.
  • Permet de publier, rechercher et localiser des services sur le web.
